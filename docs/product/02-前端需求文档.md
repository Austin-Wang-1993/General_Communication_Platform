# 通用英语对话练习 · 网页 Demo 前端功能需求

> **文档版本**：v0.2.6  
> **更新时间**：2026-05-14  
> **适用范围**：本文档描述 **C 端网页 Demo** 的页面、导航、控件状态与验收口径；与同目录 **`01-产品需求文档.md`**（下称「场景包 PRD」，当前对齐 v0.5.5）在字段语义上对齐。工程落地参见 **`../engineering/01-技术方案.md`**（当前对齐 v0.3.7）。  
> **关键约束**：① 本 Demo **移动端优先**——主要测试设备为**手机浏览器**（用户横/竖屏 9:16 ~ 4:3），所有页面布局须**先以 375~430 dp 宽度可读、可点为基线**，再向桌面端响应式放大。② **章节自由推进**：用户可从章节列表任意挑节练习，不强制线性顺序（已对齐 PRD v0.5.0）。③ **进入小节即开场**：用户进入任意未对话过的小节时，由 NPC 自动发起首条对话。  
> **非目标**：账号体系、支付、安装态引导、生产级权限与埋点规范。

## 2. 全局规则（跨页面）

### G1 调试模式

- **入口**：P1、P2、P2.1、P2.3、P2.5、P3 等 C 端页面右上角「调试模式」常驻（Demo 不设权限开关）。  
- **行为**：点击后导航至 **独立调试页 DBG**；不嵌入当前 C 端 Tab 壳层。  
- **返回**：由调试页提供返回上一 C 端或首页的策略（技术实现任选，需在实现层一致）。

### G2 创作向导顶栏三步进度（线框「三 Tab」对应实现）

- 文案顺序固定：**基本描述** → **框架生成** → **生成世界**（与线框一致；其中「框架生成」对应进入 P2.2/P2.3 链路，「生成世界」对应 P2.4/P2.5）。  
- **禁止**通过点击非当前步跳转到未完成步骤；点击时 **无导航** 或 **Toast**：「请先完成当前步骤」。  
- 高亮仅反映当前真实所在步骤。

### G3「重置创作」硬规则（与场景包 PRD 一致）

当某场景包已存在 **`story_framework`（或等价「框架已生成」标志）为真** 时，若用户在 **P2.1** 对**任意**表单字段发生编辑并再次点击 **「下一步」** 进入 P2.2：

- **必须**删除该包下的：`story_framework`、角色表（§6.3 产出）、以及**尚未进入运行期**的全部 §6.4 / §6.5 分区内容（若曾生成）。  
- **不得**影响其他 `scenario_id` 的场景包。  
- UI 可选：首次检测到「将重置」时弹出二次确认（产品可选）；若不做确认，也须在需求上接受「静默覆盖」。

### G4 P2.4「取消」清除范围

用户于 P2.4 点击「取消」回到 P2.3 时：

- **必须**清除该包下**全部**已落库的 `section_narrative` 与 `section_mission`（及与本次世界渲染任务绑定的中间态）。  
- **保留** P2.3 已展示的框架与人物清单（除非用户随后走 G3 从 P2.1 再次重置）。

### G5 P2.5 之后入口约束

- 用户进入 **P2.5** 后，**不提供**返回 **P2.1 / P2.3** 的向导路径（无「上一步」、无指向框架预览的底部导航）；顶栏「← 返回首页」与底栏「返回首页」一致，仅回到 **P2** 场景清单。  
- **当前 Demo 不提供**从运行期（P3）回到该包 P2.1 的入口；若未来增加「编辑场景信息」，须走「新建场景包」或在独立需求中明确「触发 G3 重置创作」。

### G6 P3「回答提示」（R1）

- **仅当**系统状态为「等待用户回复」且已存在可对齐的 `target_turn_id`（语义见场景包 PRD §6.7）时：允许打开弹窗并请求/展示建议。  
- **否则**：按钮 **置灰** 或点击后 Toast：「当前无需回复建议」。  
- 弹窗可关闭；关闭不改变对话状态。

### G7 P3「总结分析」（R2）

- 用户每次点击：触发一次 §6.8 对应流程；Loading 文案固定为：**「正在生成中，请稍后」**。  
- **成功**：新结果 **覆盖** 同小节已存在的成功分析内容（展示层与持久层策略一致）。  
- **失败**：展示可读错误；**保留**上一份已成功内容（若有）。  
- **禁止**后台静默定时自动生成。

### G8 P3 底部「选择信息接收人」

- **仅约束**用户**即将发送**的一条消息的 `recipient_id`。  
- **不**约束模型生成 NPC 台词时的说话对象编排（由对话生成与 §6.6.4 等规则负责）。  
- **候选范围**：仅限**当前小节** `section_narrative.appearing_npc_ids` 中的 NPC（不含 `user`）；顺序与角色表展示顺序由技术约定且须**稳定**（多 NPC 时「默认第一个」指该稳定顺序之首项）。  
- **仅 1 名 NPC 在本节出场**：进入该小节或切换到该节后，**默认选中**该 NPC；发送时 `recipient_id` **必须**为该 NPC。入口可展示为「当前接收人：{姓名}」；点击「选择信息接收人」时，可展示**仅含一项**的浮层，或省略浮层（实现二选一，须在整应用内一致）。  
- **多名 NPC 在本节出场**：进入或切换到该节后，**默认选中**上述稳定顺序中的**第一位** NPC；用户点击「选择信息接收人」须弹出**浮层**（底部动作表或居中模态均可），列出本节全部出场 NPC 供点选；用户选定后浮层关闭，底部回显当前选中 NPC；点击「发送」时 `recipient_id` **必须**为用户当前选中的 NPC。

### G9 P3a 章节自由切换（对齐 PRD §5.6）

- 用户在 **P3a** 中可选择 **任意**合法 `(chapter_id, section_id)` 并「确认进入」，**不**受任何"必须先完成前一节"的约束。  
- 允许的切节模式（与 PRD §5.6 完全一致）：
  - **顺序前进**：从 `S[i]` 切到 `S[i+1]`；
  - **跳跃前进**：从 `S[i]` 直接跳到 `S[j]`（`j > i+1`），不必通关前面的节；
  - **回头复练**：从 `S[i]` 切回 `S[t]`（`t < i`），并在该节**继续追加**新的 `turn`（不清空历史）。
- 确认后：更新运行指针为该节；P3 聊天区切换为**该节**上下文，历史回合按 `created_at` 升序加载并展示。  
- **若目标小节当前 `turn` 数为 0**：进入后立即触发 **G11 进节自动开场**；用户先看到 NPC 主动发起的开场白，然后才轮到自己开口。
- 切节本身**不**清空任何小节的历史对话；切节前的 P3 输入框草稿**丢弃**（见 P3-07-03）。
- "清空本节对话记录"动作**首版不实现**（若未来产品决定支持，将作为独立按钮在 P3 增加并触发后端清空接口）。

### G10 长耗时创作页的「分步进度」展示（P2.2、P2.4）

创作流水线单次等待可能较长，须在 **P2.2、P2.4** 主区提供 **「当前正在生成什么」** 的显性反馈，降低死等体感。

**布局要素（两页通用）**

1. **主标题行**（固定或弱变化）：P2.2 如「场景框架渲染中」；P2.4 如「专属世界渲染中」。  
2. **当前进度文案**（**必填展示位**）：单行或多行短文案，概括**此刻正在执行的子步骤**；推荐句式前缀 **「正在生成…」** / **「正在处理…」**，内容与后端/编排下发的阶段信息一致（见下）。  
3. **Loading 动效**（既有）：与进度文案同时存在。  
4. **「取消」**（既有）：行为不变。

**进度文案的数据来源（产品约束）**

- **优先**：由接口轮询、WebSocket / SSE 或任务状态中的 **`current_step_label`（或语义等价字段）** 驱动；文案随阶段变化**实时更新**（例如 P2.2：「正在生成剧情框架…」→「正在生成参与角色…」；P2.4：「正在生成第 1 章第 1 节场景…」→「正在为第 1 章第 1 节生成任务目标…」→下一小节…）。具体阶段枚举由**技术方案**与后端编排对齐本文档的**展示意图**即可。  
- **降级**：若短期内无阶段字段，前端**必须**仍显示静态兜底句（P2.2：**「正在生成场景框架，请稍候」**；P2.4：**「正在生成各小节内容，请稍候」**）；不得出现无说明的纯白屏等待。  
- **诚实原则**：不得用**虚假**进度百分比欺骗用户；若仅能提供粗粒度阶段，只用文案切换即可。

### G11 进节自动开场（对齐 PRD §6.6.5）

**触发**：以下入口都可能触发本规则：

1. P2.5「直接开启对话」 → 首次进入 P3 且默认指针为 `S[0]` 时，若 `S[0]` 的 `turn` 数为 0；
2. P2「进入场景」 → 加载到 P3 当前指针所指小节，若该节 `turn` 数为 0；
3. P3a「确认进入」 → 切换到目标小节，若目标小节 `turn` 数为 0；
4. 用户刷新 / 重新打开 P3 落回当前指针，若当前指针所指小节 `turn` 数为 0。

**前端体验要求**：

- 进入小节后，**先**渲染聊天空态骨架（顶部背景介绍栏可见、底部输入区不可用），**再**自动调用后端"生成本节首条回合"接口；用户**不需要**手动点任何按钮。  
- 等待过程中聊天区展示 **NPC typing 指示**（"对方正在输入…" / 三个跳动小圆点等），底部输入区**禁用**（灰态 + placeholder「等待对方先开口…」），"发送"按钮禁用。  
- 收到首条回合后，**追加**一条 NPC 气泡，typing 隐藏，底部输入区**启用**，"选择信息接收人"按 **G8** 规则设置默认值。  
- 若首条生成失败：聊天区展示明确错误说明 + **「重试本节开场」**按钮；用户重试期间输入区仍禁用；重试成功后回归正常流程。  
- 若该节已有 `turn ≥ 1`，**不**触发本规则，直接展示历史并启用输入区。

### G12 移动端优先与响应式（全产品默认）

本 Demo **主要测试设备为手机浏览器**，所有页面：

- **基线视口**：375 dp × 667 dp（iPhone SE 类）到 430 dp × 932 dp（iPhone 14 Pro Max 类）；
- **基本字号**：正文 14~16 sp；标题 18~22 sp；按钮可点区域 ≥ 44 dp（Apple HIG）；
- **关键控件位置**：顶部导航与"返回首页"在 56 dp 状态栏下；P3 底部输入区与"发送"按钮**贴底固定**，唤起软键盘时输入区跟随键盘上浮；
- **桌面端响应式**：≥ 768 dp 时主区可居中并限制最大宽度（如 640 dp），两侧留白即可，不必专门做 PC 端布局；
- **避免**：依赖 hover 的交互（手机无 hover）、鼠标右键菜单、依赖窗口拖拽的弹层；
- **聊天区滚动**：自动滚到底部，新消息出现时优先停在最新；如果用户已向上滚动浏览历史，则**不**自动跳到底部，仅显示"回到底部"小浮标。

##### F-P2-01 展示场景包列表

### 3.10 P3 对话练习

#### 页面作用与布局

- **作用**：运行期主界面；展示当前小节对话；提供背景/提示/复盘/章节列表入口。

##### F-P3-07 聊天记录展示、聊天气泡与 typing

**聊天记录（本节全量）**

- **数据源**：当前运行指针 `(chapter_id, section_id)` 下，服务端返回的该节全部 `turn`（与场景包 PRD §6.6.3 一致），按 **`created_at` / 服务端约定的时间升序** 排列。  
- **进入 P3 / 从 P2.5「直接开启对话」进入**：加载并展示该节已有全部回合；无历史时展示空态（引导用户开口或等待 NPC）。  
- **从 P3a「确认进入」切换小节**：清空主区列表后重新拉取**新小节**的全部回合并展示；输入区草稿按 **P3-01-02** 同级策略：**切换小节时丢弃未发送草稿**（须写入本文档或本表故事，见下）。  
- **用户发送成功**：本地或刷新后**立即**将新回合追加进聊天记录（或整表刷新），用户能在列表中看到**自己刚发**的一条，且顺序正确。  
- **NPC 回复写入成功**：同上，新气泡出现在记录末尾（或正确序位）。  
- **范围**：主聊天区**仅展示当前小节**记录，不混入其他小节气泡（除非技术显式做「折叠历史」——本 Demo **不要求**）。
- **气泡抬头（与 PRD 回合字段对齐）**：  
  - **NPC 回合**（`speaker_id ≠ user`）：正文上方展示**说话人**显示名（与现有线框一致）。  
  - **用户回合**（`speaker_id = user`）：正文上方展示 **「{发出方显示名} → {接收方显示名}」**，其中接收方为当条消息的 `recipient_id` 对应 NPC（即用户在 **F-P3-06** 中选定的信息接收人）；发出方一般为「你」或包内练习者显示名（与 roster / 界面约定一致）。样式与 NPC 抬头行同级（小字号、与气泡对齐：用户消息整体右对齐时抬头行**右对齐**）。

| 故事编号 | 前置条件 | 用户操作 | 预期结果 |
|----------|----------|----------|----------|
| P3-07-01 | 已进入某小节且该节已有历史回合（`turn ≥ 1`） | 打开 P3 注视列表 | 按时间升序展示**该节全部**已有用户与 NPC 消息；输入区**启用** |
| P3-07-02 | 已进入某小节且该节尚无回合（`turn === 0`） | 打开 P3 | 触发 **G11 进节自动开场**：聊天区先空、显示 NPC typing；输入区**禁用**；首条 NPC 回合到达后展示并启用输入 |
| P3-07-02b | 进节自动开场调用失败 | 等待结束 | 聊天区展示错误说明 + 「重试本节开场」按钮；输入区保持禁用；点击重试再次触发 |
| P3-07-03 | 在 P3；系统等待用户；已输入未发送草稿 | 从 P3a 确认切换到**另一小节** | **丢弃**未发送草稿；列表切换为新小节**全量**记录；若新小节 `turn === 0` 立即触发 **G11** |
| P3-07-04 | 在 P3；发送请求成功返回 | 无操作 | 聊天记录**更新**，出现新用户消息气泡（含 **P3-07-09** 抬头「发出方 → 接收方」），内容与发送框一致；输入框清空 |
| P3-07-05 | 用户发送后 NPC 轮次生成中 | 观看 | 出现 **typing** 指示（或等价「对方正在输入」） |
| P3-07-06 | NPC 回复已落库 | 观看 | 新 NPC 气泡追加至记录；typing 结束 |
| P3-07-07 | 新消息到达后列表变长 | 无操作 | 列表自动滚动至底部；若用户已向上滚动浏览历史，则**不**自动跳底，仅显示「回到底部」浮标（与 **G12** 一致） |
| P3-07-08 | 拉取本节记录失败 | 进入该节 | 展示错误态与重试入口（可选） |
| P3-07-09 | 列表中已有用户发送的回合（`speaker_id=user`） | 注视该绿色气泡 | 正文上方可见 **「发出方 → 接收方」**（显示名来自角色表 + `recipient_id`）；便于回顾「这条发给了谁」 |
