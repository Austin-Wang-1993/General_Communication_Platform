## 0. 一句话与三条主线

**一句话**：为「中文工作/学习语境为主、但需要在英文语境里拿机会」的人，提供**可自建、可沉浸练习**的行业/岗位英语场景与对话能力——不是背词表，而是在**有剧情、有 NPC、有任务**的世界里把专业英语练出来。

**三条主线**（与产品文档一致，便于对外复述）：

1. **场景包**：每个练习世界独立、可复用、可扩展（对应中台「场景包」模型）。  
2. **章节自由 + 进节即开场**：降低「面对空白对话」的心理成本，提高练习启动率。  
3. **双轨辅助**：对话主线之外，**回答提示（R1）**与**本节复盘（R2）**增强「练得会、练完知道自己哪里弱」——利于付费价值感知。

## 第一部分：为什么要做（战略叙事）

### 1.1 宏观背景：泛 AI 成熟之后的「切口」问题

- **供给端**：大模型与泛 AI 应用已高度成熟，通用「问一问、写一写」类产品供给过剩，**差异化难、获客贵**。  
- **渠道端**：大量用户触点仍被**大厂流量与超级 App**把持，独立团队若只做「更通用的 Copilot」，往往缺乏可持续的叙事与壁垒。  
- **结论**：更适合创业团队的路线是**垂直场景 + 可验证的用户结果**——在足够窄的切口上把体验与数据做深，再横向扩展。

### 1.2 场景痛点：中文语境里的「英文机会成本」

典型矛盾：

- 从业者（工程师、医护、酒店、产品/项目等）**日常业务语言是中文**，但**晋升、跳槽、出海、外资客户、英文面试**等机会要求其在**专业语境**下使用英语。  
- 学生群体在缺少实习/岗位机会时，仍需要**可感知专业度的练习载体**，而非仅教材与散点视频。  
- **痛点本质**：缺的不是「单词」，缺的是**连续语境中的表达习惯 + 反馈 + 低心理负担的重复练习环境**。

### 1.3 机会命题：垂类「沉浸式专业英语 / 模拟面试」

我们选择的垂直方向可概括为：

- **行业 / 岗位相关的沉浸式英语对话**（含 mock 面试、会议、项目沟通等变体）；  
- 配合**专业词汇与任务目标**，让用户在「像真的在工作」的对话里完成输出；  
- 与「通用聊天机器人」区分：**强结构、强场景、强反馈**（任务、章节、NPC、回合规则、R1/R2）。

### 1.4 价值逻辑：「铲子」定位与用户利益绑定

叙事上采用**铲子型产品**隐喻：

- 用户的核心动机往往是**更好的工作机会与收入**；语言能力是其中一种**高杠杆、可自主投资**的变量。  
- 若产品能帮助用户在**更短时间**内完成「中文舒适区 → 英文工作语境」的切换，本质上是在**帮助别人提高盈利概率或职业选择权**——与用户的**利益动机**一致，利于付费与传播。  
- 这与「纯娱乐型陪聊」或「纯工具型一次性生成」不同：我们强调**练习闭环**与**可重复进入的场景包**。

## 第二部分：市场观察与竞品格局（归纳）

> 本节为**方向性归纳**；具体产品名与数据需在后续版本中**补充独立竞品调研附录**（避免口头信息失真）。

### 2.1 已观察到的市场信号

- **自媒体 / 知识付费**：存在「某某行业 / 某某岗位英语面试资料」等**垂类资料**，说明需求真实且已有**可验证的付费行为**。  
- **英语学习类产品**：常见「场景化」嵌入——旅游、儿童、商务、工程师等，多为**语言学习 App 内嵌模块**（打卡、课程、对话脚本）。  
- **词库型产品**：在背单词逻辑中叠加**行业词书**（航空、机械、软件开发等），说明「**专业词汇**」已被市场教育。

### 2.2 我们的差异化（与上述形态对比）

| 维度 | 常见形态 | 我们的方向（与当前实现方向对齐） |
|------|-----------|-----------------------------------|
| 内容组织 | 课程单元 / 词书列表 | **场景包**：框架 → 世界 → 小节叙事与任务 → 多轮对话（中台 §5～§6） |
| 练习方式 | 单句跟读、脚本对话、轻互动 | **开放剧情结构下的连续对话** + **进节 NPC 主动开场**（降低启动摩擦） |
| 个性化 | 固定课包为主 | **用户自然语言描述**生成专属世界（对应 **intake → commit → Job** 链路） |
| 反馈 | 对错 / 分数为主 | **R1 当下怎么说** + **R2 本节沟通复盘**（与对话主线解耦，利于做增值服务） |
| 语言辅助 | 词典/翻译机 | **句级翻译辅助**（Demo：NPC 气泡与工具弹窗，中台 §6.9）——可演进为会员能力 |

**一句话差异**：不是「在学英语产品里加一点行业素材」，而是「**以行业/岗位任务为骨、以连续对话为肉**」的练习引擎。

## 第三部分：目标用户分层与优先级（讨论稿）

### 3.1 用户群像

| 分层 | 典型诉求 | 与产品的结合方式 |
|------|----------|------------------|
| **A. 白领 / 专业人士（建议核心）** | 跳槽、出海、外资协作、英文面试、岗位英语表达 | 行业/岗位场景包、mock 会议与面试、R1/R2 强化「可用表达」 |
| **B. 学生** | 四六级、专四专八、雅思托福、预科/留学前的学术与生活英语 | 考证/学业场景模板 + **传播属性强**（见第五部分） |
| **C. 生活 / 留学 / 家庭辅助** | 旅游、海外生活片段、子女留学准备等 | 作为**辅助场景**丰富 SKU，避免早期 SKU 过窄导致「不会用产品」 |

### 3.2 MVP 阶段的内容优先级（与产品路线对齐）

建议策略：**职业向场景为主**，学生与日常生活场景为辅——三向各做**少量高质量样本**，验证「沉浸感 + 自建场景 + 数据反馈」是否成立，再扩 SKU。

## 第四部分：产品演进路线（愿景 vs 当前仓库）

下列阶段为**商业路线图**；**与代码是否已实现**见小节末对照表。

| 阶段 | 目标 | 关键能力 | 相对当前 Demo |
|------|------|----------|----------------|
| **0. 当前** | 可演示的端到端 Demo | 场景包 CRUD、创作 Job、运行期对话、R1/R2、翻译辅助、部署脚本 | **已实现**（见 `README.md`） |
| **1. MVP+** | 可对外封闭测试 | 官方精选垂类模板库、体验与质量打磨、账号与基础计量（待定） | **部分有雏形**（用户自建场景已通；模板库需产品化） |
| **2. 社区化** | 网络效应冷启动 | 用户贡献「场景模型」、他人可引用；基础治理（审核、版权、举报） | **规划中** |
| **3. 激励化** | 供给端飞轮 | 下载量、点赞、收藏、投票等与**创作者激励**挂钩（返点/积分/分成，待定） | **规划中** |
| **4. 深度化** | 壁垒 | 垂类数据与词库沉淀、行业专家共建、**专用或精调语言模型**（成本高、远期） | **远期** |

**自建场景**：与现有「五字段 intake + 世界生成 + 自由章节」一致——用户不因官方 SKU 局限而流失。  
**社区与激励**：属于**下一阶段**；需在**版权、分成、内容安全、冷启动运营**上单独立项，不宜与 MVP 技术验证混为一谈。

## 第五部分：商业模式雏形——三类用户价值

我们希望同时捕获三类价值（可并存于不同人群）：

| 价值类型 | 行为特征 | 可能的机制方向（待验证） | 主要人群假设 |
|----------|----------|---------------------------|----------------|
| **传播价值** | 分享、邀请、裂变 | 低成本解锁：如**可保存/可创建场景包数量**与分享任务挂钩 | **学生**时间相对多、传播意愿强，且是未来职场用户 |
| **付费价值** | 为省时间/确定性付费 | 订阅 / 按场景包 / 按高级模型与 R1/R2 次数等 | **白领**时间机会成本高，为「更快切换英文工作语境」付费意愿更强 |
| **内容价值** | 生产优质场景、带动传播 | **创作者分成 / 返点**、榜单、与 KOL/KOC 利益绑定 | 学生与职场中「动手能力强、有垂类积累」的人 |

**设计原则**：用**利益绑定**让贡献者与平台同向——好内容带来好留存与好传播，而非一次性补贴买量。

## 第六部分：收入与盈利（讨论层，非财务预测）

- **收入可来自**：订阅、场景包/模板付费、企业版（席位、管理、审计）、高级模型与增值功能（R1/R2 深度报告、翻译包月等）、**社区分成**（若走通阶段 2～3）。  
- **成本大头**：LLM 调用、内容审核与运营、获客。需在技术方案中已有「单用户文件存储」等假设之上，单独做**单位经济模型**表（本备忘录不展开数字）。  
- **合规**：用户生成内容（UGC）、面试模拟涉及的个人数据、未成年人场景等，需在商业化前单独立法与产品条款清单。

## 第八部分：关键风险、假设与待决问题

1. **模型依赖**：质量、价格、政策与供应商切换预案。  
2. **UGC 与社区**：版权、抄袭、隐私、恶意内容——未做前勿过度承诺「人人可卖场景」。  
3. **从 Demo 到付费**：账号、支付、发票、退款、企业采购流程——当前**刻意未做**（见附录）。  
4. **竞品深度调研**：需沉淀为独立附录（产品名、定价、功能矩阵、用户评价样本）。  
5. **「专用大模型」**：属长期壁垒叙事，需与数据飞轮、标注成本、合规一并论证。
